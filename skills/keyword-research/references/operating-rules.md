# Правила работы банка ключей (вкл. поведение по расписанию)

Эти правила — контракт для агентов `keyword-research` и `keyword-performance-analyzer`
и для запуска по расписанию (cron / `/schedule`).

## Где живёт банк
`keywords/<тема>.json` в корне проекта (рядом с кодом проекта). Один файл на тему/локаль.
Минус-слова (negatives) хранятся в ТОМ ЖЕ файле, в массиве `negatives`.

## Поведение при ЗАПУСКЕ ПО РАСПИСАНИЮ (обязательно)
Когда суб-агент стартует по таймеру, он ПЕРВЫМ делом:
1. Заходит в проект, находит файл(ы) `keywords/*.json`.
2. Загружает: активные ключи + список **минус-слов** (negatives).
3. **НИКОГДА не предлагает и не добавляет ключи, которые есть в negatives** (их анализатор
   пометил как «не сработали»). Дедуп — на уровне `keyword_bank.py add` (он сам отсеивает).
4. Только потом расширяет/анализирует.

## Каденс (рекомендованный)
- **Ежедневно — СБОР** (дёшево): `keyword-research` добавляет новые кандидаты из
  Google Autocomplete / People-Also-Ask / related / новых запросов GSC → статус `new`.
- **Раз в неделю — ВЕРДИКТ** (нужна статзначимость): `keyword-performance-analyzer`
  тянет метрики из Meta Ads (рекламные ключи/таргет-термы) и Google Search Console
  (impressions/clicks/position) → `mark_performance.py` размечает 🏆winner / 🚫loser / ⏳active.
  Дневной вердикт ВРЕДЕН — реклама/SEO не успевают накопить данные (шум → ложные минусы).

## Что делает анализатор (что «сработало / нет»)
- **winner** — есть конверсии ИЛИ заметно высокий CTR → приоритет, и это новые **seed** для расширения.
- **loser** — набрал показы, но 0 конверсий и низкий CTR → при `--auto-negative` уходит в
  **минус-слова** (исключается из будущих кампаний и из контент-плана).
- **active** — данных мало → ждём следующей недели.

## Цикл самоулучшения
winner → seed для нового расширения (растим вокруг работающего) · loser → negatives
(не тратим бюджет повторно) · банк растёт и «умнеет» каждую неделю без ручного труда.
Источник правил ранжирования — своды Google/AI SEO проекта (если подключены).

## Связи
- Банк ↔ `seo-content-brief` / `content-article` (что писать по winner-ключам).
- Метрики ← `paid-ads-manager` (Meta Ads) + `pm-analytics` (GSC/GA4).
- Расширение ← `seo-cluster` (кластеризация по выдаче).
