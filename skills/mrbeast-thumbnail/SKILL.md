# MrBeast Thumbnail

Сборка обложек «как у MrBeast», но с упором на **доверие** (реальный скрин продукта
на фоне, а не AI-фейк) + **узнаваемое лицо**.

> **Простыми словами:** даёшь фон-скриншот, фото себя и текст — на выходе яркая
> обложка с твоим вырезанным лицом и огромными буквами.

## Когда брать
«Сделай обложку / превью», «thumbnail», «в стиле MrBeast», «вырежи лицо и вставь»,
«5 вариантов обложек», «cover под Shorts/YouTube».

## Пайплайн (3 шага)

### 1. Кадр-фон из видео (если фон — кадр ролика)
```bash
ffmpeg -ss <сек> -i "<video>" -frames:v 1 -y output/bg.jpg
```
Бери кадр с «доказательством» (дашборд с цифрами, список, график). Учитывай
поворот iPhone: ffmpeg авто-ротирует → вертикальный кадр станет 1080x1920.

### 2. Вырезать фон у лица (Replicate rembg)
```bash
tools/cutout_face.sh "<фото-лица>" output/face_cut.png
```
- HEIC конвертится через `sips` автоматически (если есть).
- Выбирай фото где **смотрит в камеру** + эмоция (улыбка/удивление > нейтраль).
- Токен Replicate берётся из `REPLICATE_API_TOKEN` (env или `.env`). Без него
  скрипт понятно сообщит, что нужен ключ, и не упадёт молча.

### 3. Собрать обложку из конфига
```bash
python3 tools/build_thumbnail.py config.json
```
Конфиг — JSON со слоями (см. `tools/example_config.json`).
Для **5 вариантов** — 5 конфигов с разными `texts`/цветами/`face_dx`, рендерь циклом,
склей контактный лист и покажи одним изображением (выбор по номеру).

## ПРАВИЛА обложки (выстраданный рецепт)

1. **Фон = реальный скриншот** результата (дашборд, цифры, список) → доверие, «не фейк».
   Затемнить градиентом: темнее сверху (под текст), светлее снизу.
2. **Лицо вырезать** (rembg) и сделать «парящим» с мягкой тенью. По пояс,
   bottom-aligned, как правило **справа**, смотрит в кадр/на текст. Глаза не закрывать текстом.
3. **Бренд-слово ОГРОМНОЕ** — жёлтый `#FFD200` + толстая чёрная обводка (12–16px).
   Вторичное — голубой `#96F0FF`.
4. **Заголовок** — белый, тяжёлый bold, обводка 6–10px. Ключевое слово — жёлтым.
5. **Строка-выгода** — зелёный `#2BE34D` (деньги/рост/«без рук»).
6. **Красный «маркерный» овал** `#ED1C24` — эмоц. акцент на ключевой фразе (как от руки,
   двойная линия). Не злоупотреблять — 1 на обложку.
7. **1–3 слова на «панч-строку».** Читаемость с миниатюры важнее деталей.
8. **Safe-zones:** текст не у самых краёв; низ (≈нижние 360px у 9:16) держать
   чище — там UI Shorts (счётчики/подпись).
9. **Палитра фиксирована:** yellow/cyan/green/red/white + **всегда чёрная обводка**.
10. **Шрифт** — бандл `assets/fonts/Montserrat.ttf` (тяжёлый bold с кириллицей).
11. **Орфография:** проверяй текст перед вжиганием — на публичной обложке ошибка
    дороже, чем правка.

## Форматы
- **9:16** `1080x1920` — Shorts / Reels / TikTok cover (дефолт, если ролик вертикальный).
- **16:9** `1280x720` — обычная обложка YouTube (фон слева панелью, текст справа).

## Что НЕ делать
- Не лепить AI-фон вместо реального скрина без запроса (теряется доверие).
- Не закрывать текстом глаза/лицо.
- Не оставлять нейтральное лицо, если есть кадр с эмоцией.
- Не вжигать текст с опечаткой, не переспросив (см. правило 11).

## Конфигурация
Бренд-дефолты (цвета палитры, шрифт) читаются из `.corevia/config.json` в корне
проекта. Если файла нет — запусти `/setup`, либо задай `font`/цвета прямо в конфиге.
Ключи (`REPLICATE_API_TOKEN`) — только из env/.env.

## Связи
- Монтаж/нарезка вертикальных шортсов: скилл `video-render-vertical`.
- Слой стиля удержания для самого ролика: скилл `mrbeast-montage`.
