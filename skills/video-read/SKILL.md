# Video Read — раскадровка и считывание видео

Claude не «смотрит» видео напрямую, но может **посмотреть кадры** как картинки. Этот скилл
вытаскивает кадры из видео → ты/Claude открываешь их (Read) → понятно, что на видео.

## Требования
ffmpeg (проверка/установка — `bash scripts/setup-video.sh`). Скрипт берёт ffmpeg из
переменной `FFMPEG` или из PATH.

## Как пользоваться
```bash
# 12 равномерных кадров + контактный лист (обзор всего видео одной картинкой):
bash scripts/extract_frames.sh --video путь/к/video.mp4 --out output/frames --count 12 --contact

# или кадр каждые 5 секунд:
bash scripts/extract_frames.sh --video video.mp4 --out output/frames --every 5
```
Затем **открой кадры** (`output/frames/frame_*.jpg`) или контактный лист
(`output/frames/contact_sheet.jpg`) инструментом Read — Claude увидит содержимое и опишет
сцены/текст/объекты.

## Когда что
- **--count N** — быстрый обзор всего ролика (равномерно по длительности). Дефолт 12.
- **--every S** — детально (кадр каждые S секунд) для коротких/важных фрагментов.
- **--contact** — собрать все кадры в одну картинку-сетку (удобно «прочитать» одним Read).

## Зачем это в офисе
- Понять, что на присланном видео (брифинг, реф, конкурент).
- Подготовить раскадровку для монтажа (`video-render-vertical`, `mrbeast-montage`).
- Найти момент/кадр для обложки (`mrbeast-thumbnail`).

## Что НЕ делает
- ❌ Не «смотрит» видео в реальном времени — работает по извлечённым кадрам.
- ❌ Не хранит ключи/пути — ffmpeg из env/PATH, вывод в `output/`.

## Связи
`video-render-vertical` (монтаж), `mrbeast-montage` (cut-план), `mrbeast-thumbnail` (обложка),
`scripts/setup-video.sh` (установка ffmpeg).
