# Ad Creative Studio — гибридная фабрика рекламных креативов

**Главный принцип:** ИИ-модели ломают текст на картинке (особенно кириллицу:
«ЭКСПОРТ» → «ЭКСРОРТ»). Поэтому **ИИ/стоки дают только ФОН, а текст печатается
кодом** настоящим шрифтом. Это даёт премиум-картинку + 100% читаемый текст на
любом языке. Подробно — `references/recipe.md`.

## Когда брать
Нужны рекламные/контентные креативы (посты, баннеры, обложки) под бренд проекта.
Этот скилл — про статичные рекламные изображения «фон + текст» (не видео).

## Стек (Pillow + ffmpeg-не-нужен)
- Фон: **Pexels** (бесплатные фото, commercial-free) — `scripts/fetch_stock.py`.
  Альтернатива — ИИ-фон Ideogram через Replicate (см. recipe), но БЕЗ текста в промте.
- Текст: **Montserrat** (`assets/fonts/Montserrat.ttf`, Cyrillic, веса до Black) — PIL.
- Ключи: `PEXELS_API_KEY` и `REPLICATE_API_TOKEN` берутся из переменной окружения
  или `.env` (не хардкодить в коде). Pillow — стандартный пакет.

## Workflow
### 1. Бриф → концепты
Возьми/составь бриф (концепт + копи + визуал-дирекшн + UTM). Шаблон формата —
`references/brief-to-config.md`. Если бриф уже есть в проекте — используй его.

### 2. Фоны (по одному на концепт)
```bash
export PEXELS_API_KEY=...    # из .env проекта
python3 scripts/fetch_stock.py --query "modern office bright" --orientation square --index 1 --out output/bg/bg_2.jpg
```
Подбирай query под концепт; orientation: square→1:1, portrait→9:16.

### 3. Конфиг креативов (JSON) → рендер
Собери `creatives.json` (схема в `references/brief-to-config.md`), затем:
```bash
python3 scripts/overlay.py creatives.json   # → final_<id>.png в out_dir
```
- `style: "clean"` — премиум-минимализм (дефолт для B2B-бренда).
- `style: "mrbeast"` — жирный Black + чёрная обводка + `accent_word` цветом (кликбейт,
  для верха воронки / охватных постов). Стресс-тест: обводка делает текст читаемым на
  любом фоне.

### 3.5. Лицо/портрет (опциональный MrBeast-слой)
Креатив может включать ВЫРЕЗАННОЕ лицо (как обложки MrBeast). Опционально:
вырежи фон у портрета любым rembg-инструментом (например, модель rembg на
Replicate — ключ `REPLICATE_API_TOKEN` из `.env`), получи PNG с прозрачным фоном,
затем в `creatives.json` у креатива добавь: `"face": "output/face.png"` (+ опц.
`"face_side": "right"|"left"`, `"face_scale": 0.92`). overlay.py сам впишет лицо с тенью.
Связка face + `style:"mrbeast"` = рекламный креатив/обложка «как у МрБиста».

### 4. UTM + апрув
- Размечай ссылки UTM-генератором (любой общий скрипт сборки utm-параметров).
- Складывай готовые файлы в `output/` текущего проекта (статус draft).
- **Публикацию НЕ запускай** без апрува владельца кампании.

## Что НЕ делать
- ❌ Не давать ИИ рисовать ТЕКСТ (ломается) — только фон, текст overlay'ем.
- ❌ Не хардкодить ключи — тянуть из env / `.env`.
- ❌ Не публиковать без апрува.
