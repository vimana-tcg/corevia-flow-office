# 🎬 MrBeast-монтаж (вирусное удержание)

Слой **стиля удержания** поверх готового конвейера коротких видео. Учит резать так,
чтобы зритель не пролистал. Работает локально, без серверов.

## Когда брать

Нужен ролик «чтобы залетал»: показываешь кейс/что сделал, нужен динамичный монтаж
в стиле MrBeast — крючок, быстрые резы, зумы, звуки, b-roll.

## Архитектура (что откуда)

```
                ┌─────────────────────────────────────────┐
ЗАПИСЬ ──────►  │  транскрипция → Whisper транскрипт        │
(screencast/    └───────────────┬─────────────────────────┘
 селфи)                         ▼
                ┌─────────────────────────────────────────┐
                │  🧠 tools/shotlist_gen.py                │  ← МОЗГ (этот скилл)
                │  транскрипт → cut-план MrBeast            │
                │  (hook→stakes→proof→loop→cta,            │
                │   зум, sfx, b-roll, тайминг ≤3с)         │
                └───────────────┬─────────────────────────┘
                                ▼
                ┌─────────────────────────────────────────┐
                │  ✅ tools/retention_check.py             │  ← оценка вирусности
                │  балл 0-100 + что починить               │
                └───────────────┬─────────────────────────┘
                                ▼  (≥80 баллов)
                ┌─────────────────────────────────────────┐
                │  🎬 video-render-vertical / montage.py   │  ← РУКИ (рендер)
                │  ffmpeg: 9:16, субтитры, зум, звук        │
                └───────────────┬─────────────────────────┘
                                ▼
                  output/<тема>/final_polished/
```

**Простыми словами:** мозг (этот скилл) пишет план «где резать, где зум, где
звук». Руки (движок рендера) собирают по плану. Проверка ставит оценку до рендера.

## Что НЕ входит

- ❌ Авто-публикация — решает пользователь
- ❌ Длинный YouTube 16:9 — отдельный движок
- ❌ Генерация аватара — отдельный скилл

## Workflow

### 1. Получи транскрипт
Если у записи ещё нет транскрипта — прогони её через Whisper (word-level). Нужен
JSON с `segments[]` (желательно с `words[]`).

### 2. Сгенерируй cut-план (мозг)
```bash
python3 tools/shotlist_gen.py \
  <transcript.json> --target 35 --platform reels --pretty > cutplan.json
```
Платформы: `reels` | `tiktok` | `shorts` | `youtube_long`.

### 3. Проверь вирусность
```bash
python3 tools/retention_check.py cutplan.json
```
Цель — **≥80/100**. Ниже — читай подсказки «что починить», правь сценарий/тайминг.

### 4. Отдай движку рендера
Передай `cutplan.json` движку `video-render-vertical` как cut plan (поле `visuals`/
`segments`). Он применит зум/sfx/субтитры. Источник звуков — см. ниже.

### 5. QA и доставка
Проверь финал → `output/<тема>/final_polished/`.

## Знания (references/)

- **`retention-dna.md`** — 12 правил удержания + как применять к любому контенту.
  Это «учебник стиля» — читай, если хочешь понять *почему* залетает.
- **`hook-bank.md`** — 9 формул крючков + ре-крючки + CTA-банк.

## Инструменты (tools/)

| Файл | Что делает |
|---|---|
| `shotlist_gen.py` | Транскрипт → cut-план в стиле MrBeast (детерминированный, stdlib) |
| `retention_check.py` | Оценивает cut-план баллом 0-100 + конкретные фиксы |
| `editing_schema.json` | Схема cut-плана (сцены, зум, sfx, b-roll) |

## Звуковая палитра (для движка рендера)

| sfx в плане | Когда | Звук |
|---|---|---|
| `whoosh` | на резах/переходах | свуш |
| `ding`/`pop` | на цифрах и акцентах | дзынь |
| `boom` | итог/CTA/раскрытие | суб-бас |
| `silence` | 0.5с ПЕРЕД раскрытием | тишина (приём ожидания) |

⚠️ Библиотека звуков не вшита — подключи свободные паки (freesound/mixkit).
До этого монтажёр ставит звуки вручную по плану.

## Конфигурация
Бренд-дефолты (цвета, шрифт, голос, имя) читаются из `.corevia/config.json` в корне
проекта. Если файла нет — запусти `/setup` для создания, либо передай параметры явно.

## Связи
- **Использует движок рендера:** `video-render-vertical` (`montage.py`)
- **Сценарии/крючки:** читай `references/hook-bank.md`
- **Карусели/посты:** та же ДНК крючков из `references/hook-bank.md`

## Антипаттерны
- ❌ Не рендерить при балле <80 — сначала починить план
- ❌ Не делать speedup >1.0x (мельтешит)
- ❌ Не публиковать автоматом
